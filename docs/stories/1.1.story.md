### TASK-1.1.7 — Corrigir testes de pacotes (enquetes_created)
- **Status:** ✅ Concluída
- **Arquivo:** `tests/unit/test_analyze_votos_packages.py`
- **Root cause:** `_process_packages()` só popula `closed_today` e `open` quando `enquetes_created` contém timestamp de criação do poll. Sem ele, ambos ficam sempre vazios.
- **Fix:** Adicionado `enquetes_created = {"poll-X": today}` em todos os testes relevantes

### TASK-1.1.8 — Corrigir test_refresh_preserves_confirmed_today
- **Status:** ✅ Concluída
- **Arquivo:** `tests/unit/test_refresh_preserve_confirmed.py`
- **Root cause:** Commit introduziu filtro de data para `confirmed_today` — pacotes com `confirmed_at` antes de `today_start` são descartados. O teste usava data hardcoded no passado sem mockar o relógio.
- **Fix:** `monkeypatch.setattr(processors, "get_date_range", lambda: fake_dates)` com datas alinhadas ao `confirmed_at` do teste

### TASK-1.1.9 — Reestruturar deploy.yml (gate real + GHCR)
- **Status:** ✅ Concluída
- **Arquivo:** `.github/workflows/deploy.yml`
- **Antes:** Job `test` (gate falso) + Job `deploy` (SSH para VPS)
- **Depois:** Job `test` (gate real) + Job `build` (build + push para GHCR)
- **Detalhes:**
  - Removido `|| echo "Tests completed with warnings"`
  - Removido job `deploy` com SSH (VPS gerenciado pelo Portainer)
  - Adicionado job `build` com `docker/build-push-action@v5`
  - Imagem publicada em `ghcr.io/V4MarcosPaulo/alana-dashboard:latest` e `sha-{hash}`

### TASK-1.1.10 — Corrigir deadlock no teste de concorrência
- **Status:** ✅ Concluída
- **Arquivo:** `tests/unit/test_sort_and_assembly_and_concurrency.py`
- **Root cause:** `asyncio.Lock()` criado fora do event loop ativo quando `TestClient` é usado sem context manager. Com 6 threads concorrentes, o lock era criado em event loops diferentes causando deadlock.
- **Fix:** Usar `with TestClient(app) as client:` — o context manager do `anyio` garante que o lock é criado no event loop correto antes das threads serem iniciadas
